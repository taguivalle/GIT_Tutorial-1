Introducción

Git es un sistema de control de versiones distribuido; GitHub es la plataforma que aloja repositorios Git y facilita la colaboración.

"Imagina que cada cambio que haces en tu proyecto queda guardado como una foto del momento."

¿Por qué usar Git?

Beneficio Qué significa para ti
Seguridad Instantánea recuperable en cualquier momento.
Colaboración Trabaja con otros sin pisar cambios.
Historia clara Cada modificación registrada con fecha y descripción.
Trabajo offline Funciona sin internet, sincroniza después.
Flexibilidad Adaptable a proyectos pequeños o gigantes.

Conceptos básicos de Git

Repositorio

Contenedor que guarda todo el historial de cambios.

git init

Commit

Instantánea del proyecto en un momento dado.

git commit -m "Mensaje"

Branch

Línea paralela de desarrollo.

git branch nueva-funcionalidad

Merge

Unir cambios de una rama a otra.

git merge nueva-funcionalidad

Staging

Zona intermedia antes del commit.

git add archivo.txt

Instalación

brew install git
sudo apt-get update && sudo apt-get install git

En Windows: descarga desde git‑scm.com.

Comandos esenciales

Comando Descripción Ejemplo
git init Crea repositorio local git init mi-proyecto
git clone Clona repositorio remoto git clone URL
git status Muestra estado de archivos git status
git add Agrega archivos al staging git add .
git commit Registra cambios git commit -m "Mensaje"
git push Envía commits al remoto git push origin main
git pull Trae cambios del remoto git pull origin main
git branch Lista o crea ramas git branch nueva
git merge Fusiona una rama git merge nueva

Flujo de trabajo típico

  1. Crear repositorio (git init)
  2. Añadir archivos (git add .)
  3. Hacer commit (git commit -m "mensaje")
  4. Enlazar remoto (git remote add origin URL)
  5. Push al remoto (git push -u origin main)

Trabajar con GitHub

  1. En GitHub, pulsa New repository
  2. Define nombre, descripción y visibilidad
  3. No marques "Initialize with a README"
  4. Copia la URL del repo (HTTPS o SSH)
  5. En tu terminal: git remote add origin <URL>
  6. Empuja tu rama: git push -u origin main

También puedes usar Fork, Pull Request y Issues para colaborar.
