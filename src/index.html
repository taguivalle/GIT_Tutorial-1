Introducción

Git es un sistema de control de versiones distribuido; GitHub es la plataforma que aloja repositorios Git y facilita la colaboración.

“Imagina que cada cambio que haces en tu proyecto queda guardado como una foto del momento. Así, siempre puedes volver atrás, comparar versiones o compartir tu trabajo con quien quieras.”

Git es precisamente eso: un sistema de control de versiones que te ayuda a llevar un registro ordenado de los cambios que haces en tus archivos, sin importar si eres alguien que recién está descubriendo la programación o un desarrollador con años de experiencia.

¿Por qué usar Git?

Beneficio Qué significa para ti
Seguridad Cada vez que “guardas” tu trabajo, Git crea una instantánea que puedes recuperar en cualquier momento. Si algo sale mal, volver al estado anterior es tan fácil como deshacer un paso.
Colaboración Puedes trabajar con otras personas en el mismo proyecto sin pisarse los cambios. Git combina las contribuciones de todos y muestra claramente quién hizo qué.
Historia clara Cada modificación queda registrada con una breve descripción y la fecha. Con solo mirar la historia, puedes entender cómo evolucionó el proyecto.
Trabajo offline No necesitas estar conectado a internet todo el tiempo. Puedes crear, modificar y revisar versiones en tu propia máquina y sincronizarlas después.
Flexibilidad Desde proyectos pequeños de una sola página web hasta gigantescos repositorios con miles de archivos, Git se adapta a cualquier escala.

Conceptos básicos de Git

Repositorio

Contenedor que guarda todo el historial de cambios del proyecto. 

git init

Commit

Instantánea del proyecto en un momento dado. 

git commit -m "Añadir README"

Branch (rama)

Línea paralela de desarrollo que permite trabajar de forma aislada. 

git branch nueva-funcionalidad

Merge

Unir cambios de una rama a otra. 

git merge nueva-funcionalidad

Staging area

Zona intermedia donde se preparan los archivos antes del commit. 

git add archivo.txt

Instalación

brew install git
sudo apt-get update && sudo apt-get install git

En Windows: descarga el instalador desde git‑scm.com.

Comandos esenciales

Comando Descripción Ejemplo
git init Crea un nuevo repositorio local. git init mi-proyecto
git clone Clona un repositorio remoto. git clone https://github.com/usuario/repo.git
git status Muestra el estado de los archivos. git status
git add Agrega archivos al staging area. git add .
git commit Registra los cambios. git commit -m "Mensaje"
git push Envía commits al remoto. git push origin main
git pull Trae y fusiona cambios del remoto. git pull origin main
git branch Lista o crea ramas. git branch nueva-funcionalidad
git merge Fusiona una rama en la actual. git merge nueva-funcionalidad

Flujo de trabajo típico

  1. Crear repositorio (git init).
  2. Añadir archivos (git add .).
  3. Hacer commit (git commit -m "mensaje").
  4. Crear repositorio remoto en GitHub y enlazarlo (git remote add origin …).
  5. Push al remoto (git push -u origin main).
  6. Crear ramas para nuevas funcionalidades, trabajar, mergear, etc.

Trabajar con GitHub

Pasos básicos para crear un repo remoto:

  1. En GitHub, pulsa New repository.
  2. Define nombre, descripción y visibilidad.
  3. No marques “Initialize with a README”.
  4. Copia la URL del repo (HTTPS o SSH).
  5. En tu terminal: git remote add origin <URL>
  6. Empuja tu rama principal: git push -u origin main

También puedes usar Fork, Pull Request y Issues para colaborar.
